### Lección 1: Primer programa en Java (7 minutos)

- **Descripción**: Qué es la programación, qué es Java y qué ocurre realmente al presionar Run: tu primer programa, la estructura mínima de una clase y `System.out.println`.
- **Ver diapositivas**: [Google Slides](https://docs.google.com/presentation/d/1sprmNEi_l7CcgXUW5FXiSdLdRUQjF06H-bfKQ30MYj0/edit?usp=sharing)
- **Conclusiones clave**: Un programa Java necesita una clase y un método `main` como punto de entrada; `System.out.println` muestra texto en consola y OnlineGDB permite ejecutar Java sin instalar nada.
- **Práctica**: [Código inicial](https://onlinegdb.com/MSGkbKeOb) 👈 ¡Inicio inmediato! · [Solución completa](https://onlinegdb.com/H6-MG2_jr)

<br>

### Lección 2: Variables y tipos de datos (9 minutos)

- **Descripción**: Qué es una variable, los tipos primitivos de Java (`int`, `double`, `boolean`, `String`), cómo declararlos correctamente y la convención camelCase para nombrarlos.
- **Ver diapositivas**: [Google Slides](https://docs.google.com/presentation/d/1DMUItfF_qLu5PUPS6NUGM_Ll2nKlG6adixuTCp2YK7s/edit?slide=id.p1)
- **Conclusiones clave**: Las variables guardan datos con un tipo fijo (`int`, `double`, `boolean`, `String`); declarar correctamente y nombrar con camelCase evita errores de compilación y hace el código legible.
- **Práctica**: [Código inicial](https://onlinegdb.com/YfvQXps3Q) 👈 ¡Inicio inmediato! · [Solución completa](https://onlinegdb.com/EwwhQb_U9)

<br>

### Lección 3: Condiciones y bucles (10 minutos)

- **Descripción**: Cómo un programa toma decisiones con `if` y `else`, los operadores de comparación, y cómo repetir instrucciones con el bucle `for` sin duplicar código.
- **Ver diapositivas**: [Google Slides](https://docs.google.com/presentation/d/1ALH1pRKjewmX0ygyONVt7DwzcqxJeEGRN0p28dCHrG8/edit?slide=id.p2)
- **Conclusiones clave**: Los operadores de comparación siempre devuelven `true` o `false`; `if`/`else` permite elegir caminos distintos y el bucle `for` repite instrucciones sin duplicar código.
- **Práctica**: [Código inicial](https://onlinegdb.com/FUVtXESMU) 👈 ¡Inicio inmediato! · [Solución completa](https://onlinegdb.com/HxXhlHg1V)

<br>

### Lección 4: Métodos y entrada por consola (10 minutos)

- **Descripción**: Cómo agrupar instrucciones en métodos reutilizables, la diferencia entre parámetros y argumentos, y cómo leer datos del usuario con `Scanner`.
- **Ver diapositivas**: [Google Slides](https://docs.google.com/presentation/d/1IPBxFwuU-O2OF1JNLGSi1J6Ta6m7MVzgdIubuaqNOGM/edit?slide=id.p1)
- **Conclusiones clave**: Los métodos dividen el código en partes reutilizables con una sola responsabilidad; `Scanner` permite leer datos que el usuario escribe en consola.
- **Práctica**: [Código inicial](https://onlinegdb.com/FKkblaXeZ) 👈 ¡Inicio inmediato! · [Solución completa](https://onlinegdb.com/scclTR8f2)

<br>

### Lección 5: Clases y objetos (12 minutos)

- **Descripción**: Introducción a la Programación Orientada a Objetos: cómo definir una clase, crear objetos a partir de ella y usar constructores para inicializarlos.
- **Ver diapositivas**: [Google Slides](https://docs.google.com/presentation/d/1raVBHI2qWbQCTI_lIZZjC_1B1Wn4pghPyYe1Qa-oe00/edit?slide=id.p1)
- **Conclusiones clave**: Una clase define la estructura de un objeto (datos y comportamiento); los constructores inicializan los objetos al crearlos con `new`.
- **Práctica**: [Código inicial](https://onlinegdb.com/ByDi4BB5V) 👈 ¡Inicio inmediato! · [Solución completa](https://onlinegdb.com/sbQ0jDP3x)

<br>

### Lección 6: Encapsulación y proyecto final (12 minutos)

- **Descripción**: Protege los datos de tus clases con encapsulación, getters y setters, y combina todo lo aprendido en un proyecto final integrador.
- **Conclusiones clave**: La encapsulación protege los datos de una clase mediante atributos privados y métodos getter/setter; un proyecto integrador combina clases, objetos y lógica de control en un solo programa funcional.
- **Práctica**: [Código inicial](https://onlinegdb.com/5iZQLdbuT) 👈 ¡Inicio inmediato! · [Solución completa](https://onlinegdb.com/7kkveYhAi_)

<br>

## Resumen

Este curso de aproximadamente 1 hora introduce a estudiantes de secundaria a la programación y a la Programación Orientada a Objetos (POO) usando Java. **No requiere descargas ni instalaciones.** Solo abre tu navegador web.

**Duración total**: ~60 minutos  
**Público objetivo**: Estudiantes de 12 a 17 años sin experiencia en programación  
**Prerrequisitos**: Ninguno  
**Herramientas necesarias**: Solo tu navegador web (Chrome, Firefox, Safari, Edge)

---

## Público Objetivo

- **Edad**: Estudiantes de 12 a 17 años.
- **Experiencia**: Sin experiencia previa en programación o desarrollo de software.
- **Prerrequisitos**: Solo necesitan saber usar un navegador web moderno (Chrome, Firefox, Safari, Edge).
- **Estilo de aprendizaje**: Preferencia por resultados interactivos rápidos, explicaciones concisas con analogías y laboratorios guiados en OnlineGDB.

## Orden recomendado

1. Leer `course-plan.md`.
2. Preparar el entorno con `docs/online-editor-guide.md`.
3. Estudiar cada lección desde `docs/lessons/`.
4. Importar o copiar el código de `starter-files/` en OnlineGDB para trabajar la práctica.
5. Comparar la práctica terminada con `completed-examples/`.
6. Resolver el quiz de autoevaluación correspondiente al final de cada lección.

>[!NOTE]
> **Cada carpeta de `starter-files` y `completed-examples` está pensada para copiarse y ejecutarse como un proyecto independiente. Por eso varias lecciones tienen una clase `Main` en el default package. Si un IDE local intenta compilar todas las carpetas a la vez, puede marcar clases duplicadas; la validación correcta es compilar cada lección por separado.**
